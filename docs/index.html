1 Install

Choose your platform below and run the command as Administrator / root.

PowerShell — run as Administrator
irm https://raw.githubusercontent.com/Lexaplus/CustomDns/main/client-setup/windows-doh-setup.ps1 | iex

This single command: installs the server certificate, registers the DoH template, and sets DNS on all active adapters. 1.1.1.1 is added as a fallback — if the private server is unreachable, DNS automatically falls back to Cloudflare so your internet keeps working.

If execution policy blocks it:

powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/Lexaplus/CustomDns/main/client-setup/windows-doh-setup.ps1 | iex"

To undo everything:

.\windows-doh-setup.ps1 -Uninstall
PowerShell — run as Administrator
irm https://raw.githubusercontent.com/Lexaplus/CustomDns/main/client-setup/windows10-doh-setup.ps1 | iex

This single command: installs the server certificate, downloads and installs dnscrypt-proxy as a Windows service, configures it to forward all DNS over HTTPS to the private server, and sets DNS on all active adapters to 127.0.0.1. 1.1.1.1 is added as a fallback — if the private server is unreachable, DNS automatically falls back to Cloudflare.

If execution policy blocks it:

powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/Lexaplus/CustomDns/main/client-setup/windows10-doh-setup.ps1 | iex"

To undo everything:

.\windows10-doh-setup.ps1 -Uninstall
Terminal
networksetup -setdnsservers Wi-Fi 159.89.25.145

Replace Wi-Fi with your interface name if different (e.g. Ethernet).

Note: Some ISPs (especially in Iran) hijack UDP port 53 and return fake results. Use +tcp to test reliably.

Verify:

scutil --dns | grep nameserver
dig google.com @159.89.25.145 +tcp

To revert:

networksetup -setdnsservers Wi-Fi "Empty"
systemd-resolved
resolvectl dns eth0 159.89.25.145
resolvectl flush-caches

Replace eth0 with your interface (ip link to list).

Or edit /etc/resolv.conf directly:

echo "nameserver 159.89.25.145" | sudo tee /etc/resolv.conf

2 Verify

Run the verification script to confirm all 6 checks pass.

PowerShell — run as Administrator
irm https://raw.githubusercontent.com/Lexaplus/CustomDns/main/client-setup/windows-doh-verify.ps1 | iex
ISPs in some regions (e.g. Iran) hijack UDP port 53. Always use +tcp or the DoH test below — plain dig without +tcp may return 198.18.x.x even when the server is working.
Terminal — TCP DNS test
dig google.com @159.89.25.145 +tcp

You should see an ANSWER SECTION with a real IP. If it times out, your IP is not yet on the allowlist.

Or test DoH directly (bypasses ISP entirely):

Python 3 — DoH test
python3 -c "
import urllib.request, ssl, base64
q = b'\x00\x00\x01\x00\x00\x01\x00\x00\x00\x00\x00\x00\x06google\x03com\x00\x00\x01\x00\x01'
url = 'https://159.89.25.145:8443/dns-query?dns=' + base64.urlsafe_b64encode(q).rstrip(b'=').decode()
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
req = urllib.request.Request(url, headers={'accept': 'application/dns-message'})
with urllib.request.urlopen(req, context=ctx) as r:
    data = r.read()
    print('google.com ->', '.'.join(str(b) for b in data[-4:]))
"

3 Troubleshooting

  • ⏳
    DNS times out / no answer
    Your IP is not on the allowlist. Contact the admin to add it.
  • 🔒
    Certificate error in browser
    Re-run the setup script as Administrator — it installs the cert into the system trust store.
  • ⚠️
    198.18.x.x returned instead of real IP
    This is AdGuard's NXDOMAIN-map address. Your IP is not yet allowlisted.
  • 🚫
    Execution policy error on Windows
    Use the -ExecutionPolicy Bypass command shown in Step 1.
  • ↩️
    Want to uninstall
    Run .\windows-doh-setup.ps1 -Uninstall as Administrator to revert all changes.
  • 🛑
    Internet stopped working after setup (Windows)
    Run in PowerShell — no admin needed:
    Set-DnsClientServerAddress -InterfaceAlias "Wi-Fi" -ResetServerAddresses
    Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ResetServerAddresses
    This restores automatic DNS from your router instantly. Then contact the admin to get your IP allowlisted before re-running the script.
  • 🌐
    dig returns 198.18.x.x even for 8.8.8.8
    Your ISP is hijacking UDP port 53. This is common in Iran and some other regions. Use dig +tcp or the Python DoH test in Step 2 to verify the server is working. The Windows DoH script is unaffected — it uses HTTPS port 8443 which bypasses the hijack.
