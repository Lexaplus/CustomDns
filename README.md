# private-dns-demo

A production-quality **Private DNS service** deployable on a single Ubuntu VPS with one bootstrap script.

> **Purpose:** demonstrate a real, hardened DNS resolver with IP allowlisting, admin panel, Telegram bot control, and observability — ready for a live Google Meet demo.

---

## Architecture

```
                        ┌─────────────────────────────────────────────────┐
                        │                  Ubuntu VPS                      │
                        │                                                  │
  Client (DNS/DoH)      │  ┌──────────────┐    ┌───────────────────────┐  │
  ──────────────:53────►│  │ AdGuard Home │    │       admin-api       │  │
  (UDP/TCP)             │  │   :53        │    │  NestJS + TypeORM     │  │
  ──────────────:8443──►│  │   :8443 DoH  │    │       :8080           │  │
  (HTTPS/DoH)           │  │   :3000 UI   │    └──────────┬────────────┘  │
  Telegram Bot          │  └──────────────┘               │               │
  ──────────────────────►  ┌──────────────┐               │               │
                        │  │   tg-bot     │───────────────┘               │
                        │  │  (grammy)    │    ┌───────────────────────┐  │
                        │  └──────────────┘    │      PostgreSQL        │  │
                        │                      │       :5432            │  │
                        │  ┌──────────────┐    └───────────────────────┘  │
                        │  │  Prometheus  │◄── node-exporter :9100        │
                        │  │    :9090     │    ┌───────────────────────┐  │
                        │  └──────────────┘    │       Grafana          │  │
                        │         │            │       :3001            │  │
                        │         └───────────►│  DNS Overview dashboard│  │
                        │                      └───────────────────────┘  │
                        │  ┌──────────────────────────────────────────┐   │
                        │  │ UFW Firewall                              │   │
                        │  │  :22   — admin IPs only                  │   │
                        │  │  :53   — allowlisted client IPs only     │   │
                        │  │  :8443 — allowlisted client IPs only     │   │
                        │  │  :3000/:3001/:8080 — admin IPs only      │   │
                        │  └──────────────────────────────────────────┘   │
                        └─────────────────────────────────────────────────┘
                                         │ optional TLS
                        ┌────────────────▼────────────────────────────────┐
                        │  Caddy (--profile tls)  :80/:443                │
                        │  Reverse proxy + auto HTTPS (when DNS_DOMAIN set)│
                        └─────────────────────────────────────────────────┘
```

**Flow:** Client DNS query → UFW checks allowlist → AdGuard Home resolves → returns answer.
Admin (API/bot) mutates allowlist → NestJS writes `/deploy/allowlist.txt` → calls `firewall_apply.sh update` → UFW rules refreshed live.

| Requirement | Notes |
|---|---|
| Ubuntu 22.04 or 24.04 VPS | DigitalOcean, Vultr, Hetzner, etc. |
| Root SSH access | `ssh root@SERVER_IP` |
| Port 53 not in use | Bootstrap disables systemd-resolved |

## Environment Variables

Copy `infra/.env.example` to `infra/.env` and fill in all values.

| Variable | Required | Description |
|---|---|---|
| `POSTGRES_USER` | yes | PostgreSQL username |
| `POSTGRES_PASSWORD` | **yes** | PostgreSQL password — change from default! |
| `POSTGRES_DB` | yes | PostgreSQL database name |
| `ADMIN_API_TOKEN` | **yes** | Secret token for `x-admin-token` header |
| `ADGUARD_ADMIN_USER` | yes | AdGuard web UI username |
| `ADGUARD_ADMIN_PASSWORD_HASH` | yes | bcrypt hash — generated by `bootstrap.sh` |
| `ADMIN_IPS` | **yes** | Comma-separated IPs always allowed for SSH + admin UIs |
| `TELEGRAM_BOT_TOKEN` | **yes** | Token from @BotFather |
| `ADMIN_IDS` | **yes** | Comma-separated Telegram user IDs |
| `ADMIN_API_URL` | yes | `http://admin-api:8080` (internal Docker DNS) |
| `GF_ADMIN_USER` | yes | Grafana admin username |
| `GF_ADMIN_PASSWORD` | **yes** | Grafana admin password |
| `DNS_DOMAIN` | optional | If set, enables Caddy TLS profile |
| `TZ` | yes | Timezone, e.g. `UTC` |

## Admin API Reference

All endpoints except `/health` require header: `x-admin-token: YOUR_TOKEN`.
Optionally pass `x-actor: your-name` for audit log attribution.

```bash
# Health check (no auth)
curl http://SERVER_IP:8080/health

# List allowed IPs
curl -H "x-admin-token: TOKEN" http://SERVER_IP:8080/allowed-ips

# Add an IP
curl -X POST \
  -H "x-admin-token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"ip":"1.2.3.4","label":"My laptop"}' \
  http://SERVER_IP:8080/allowed-ips

# Update (enable/disable or change label)
curl -X PATCH \
  -H "x-admin-token: TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled":false}' \
  http://SERVER_IP:8080/allowed-ips/1

# Remove an IP
curl -X DELETE \
  -H "x-admin-token: TOKEN" \
  http://SERVER_IP:8080/allowed-ips/1
```

## Client DNS Setup

Point your device's DNS to the server IP:

**macOS:**
```bash
networksetup -setdnsservers Wi-Fi 159.89.25.145
# Verify:
scutil --dns | grep nameserver
```

**Linux (systemd-resolved):**
```bash
resolvectl dns eth0 159.89.25.145
resolvectl flush-caches
```

**Windows 11 — one-click DoH setup (recommended):**
```powershell
# Run as Administrator — installs the self-signed cert, registers DoH template,
# and sets DNS on all active adapters
.\client-setup\windows-doh-setup.ps1

# To revert everything
.\client-setup\windows-doh-setup.ps1 -Uninstall
```
The script auto-fetches the server certificate from port 8443 at runtime, or you
can embed it as base64 in `$CERT_B64` for a fully offline installer (the cert
base64 is printed by `bootstrap.sh` after cert generation).

**Windows (plain UDP/TCP, no cert needed):**
```powershell
Set-DnsClientServerAddress -InterfaceAlias "Wi-Fi" -ServerAddresses 159.89.25.145
```

**Android / iOS:** Settings → Wi-Fi → DNS → set to `159.89.25.145`

> Your device IP must be in the allowlist first. Use the Telegram bot or Admin API to add it.

## Optional TLS (Caddy)

If you own a domain and have pointed it to the server:

```bash
# In infra/.env, add:
DNS_DOMAIN=dns.example.com

# Start with TLS profile
cd infra
docker compose --profile tls up -d
```

AdGuard UI will be available at `https://dns.example.com` with HTTP basic auth.

## Security Notes

- Ports 53 and 8443 (DoH) are **closed** by iptables DOCKER-USER chain until at least one IP is added to the allowlist
- SSH (port 22) is restricted to `ADMIN_IPS` only — set this before running bootstrap
- AdGuard setup wizard is disabled via pre-seeded `AdGuardHome.yaml`
- All secrets come from `.env` — never committed
- Firewall mutations use `flock`-based locking to prevent races
- The NestJS container runs as `node` (non-root); only `sudo /deploy/firewall_apply.sh` is allowed
